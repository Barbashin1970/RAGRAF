# Акустический детектор (`noise`)

Шумовой датчик: микрофон / гидрофон / сейсмический микрофон. На теплосетях
используется для **корреляционного поиска утечек** (полоса ~0.3..2 кГц).
В проекте **ШУМ-ИИ** (подсистема системы ГОРОД-ОМ-ИИ, см. отчёт ЦИИНГУ-2024
«Шаблон событий», §2 и §6) — для городского шумомониторинга.

## Принцип «edge-детектор, не сырая телеметрия»

> Из отчёта ЦИИНГУ-2024, §2: «датчики работают килогерцами. Всё это
> проводить через централизованную систему событий фреймворка не получится,
> если датчиков много. Значит надо данные **агрегировать на местах**,
> обрабатывать и получать **более крупные события**, и их уже заводить
> во фреймворк (например, только при превышении шума, а не просто
> значение амплитуды по тысяче раз в секунду)».

То есть в СИГМУ летят **только семантические события** от edge-детектора:
- «превышен уровень шума на узле X»
- «по акустической сигнатуре определён поезд в координатах Y»
- «утечка по корреляции на участке Z»

Сырые триплеты амплитуд (см. [raw-triplet.json](raw-triplet.json)) живут
на edge-устройстве и **не уходят за пределы локального агрегатора**.

```
датчик 24-бит триплет 1..10 кГц
   ↓ (локальная сеть, ~100 КБ/с/датчик)
edge-агрегатор / district-сервер: FFT + ML-классификация
   ↓ (HTTPS, ~1 событие/мин/датчик)
СИГМА / RAGRAF: один семантический event
```

## Прибор

- **Акустический** микрофон (воздух) — городской шум, проезды техники
- **Сейсмический** микрофон (закопан в землю) — подземные вибрации,
  ходьба, рытьё, движение тяжёлой техники
- **Гидрофон** (пьезо на трубе) — корреляционный поиск утечек
- Edge-устройство делает FFT-анализ на скользящих окнах 0.5–30 сек

## Поля payload (семантические события)

Шаблоны взяты из отчёта ЦИИНГУ-2024, §3 и §5.

### Превышение уровня шума ([noise-threshold-exceed.json](noise-threshold-exceed.json))

| Поле           | Тип    | Описание                                  |
|----------------|--------|-------------------------------------------|
| `event_type`   | string | `noise_threshold_exceeded`                 |
| `sensor_id`    | string | ID источника                               |
| `threshold`    | number | Норма в дБ (60 для жилой зоны ночью)        |
| `current_value`| number | Текущий уровень в дБ                        |
| `ts`           | int64  | Unix-timestamp мс                           |
| `pos`          | string | `"<lat>N <lon>E"`                          |

### Распознанный источник шума ([vehicle-train.json](vehicle-train.json))

ML-классификатор по акустической сигнатуре определяет тип источника
(поезд / грузовик / самолёт / пневматический молоток …). Это **результат
обработки**, а не «датчик шума выдал значение». См. отчёт §2 и §3 п.5:

```json
{ "vehicle_type": "train", "ts": ..., "coords": "<lat>N <lon>E" }
```

| Поле           | Тип    | Описание                                       |
|----------------|--------|------------------------------------------------|
| `event_type`   | string | `vehicle_detected_by_noise` (наш вариант)       |
| `vehicle_type` | string | `train` / `truck` / `airplane` / `excavator` …  |
| `confidence`   | number | (опционально) уверенность классификатора 0..1   |
| `ts`           | int64  | Unix-timestamp мс                                |
| `coords`       | string | `"<lat>N <lon>E"` — вычисленные координаты      |
| `sensor_id`    | string | (опционально) кто зафиксировал                  |

### Корреляционная утечка ([leak-acoustic.json](leak-acoustic.json))

Внутреннее использование (трубопроводы) — анализ полосы + ML-score.

## Сырые данные — для документации, НЕ для отправки

[raw-triplet.json](raw-triplet.json) показывает формат сырого замера
24-бит триплета амплитуд (3 разнонаправленных микрофона). Этот формат
**в СИГМУ напрямую не идёт** — оставляем как ссылку на исходный поток,
чтобы edge-разработчик понимал откуда берутся семантические события.

## Примеры

| Файл                                              | Уровень    | Сценарий                                         |
|--------------------------------------------------|------------|--------------------------------------------------|
| [raw-triplet.json](raw-triplet.json)             | сырой       | 24-бит триплет, edge-only, в СИГМУ не идёт         |
| [quiet.json](quiet.json)                         | агрегат     | Норма: фон 45 дБ                                  |

| [vehicle-train.json](vehicle-train.json)         | семантика  | По акустической сигнатуре — поезд (отчёт §2 / §3 п.5) |
| [leak-acoustic.json](leak-acoustic.json)         | семантика  | Утечка по корреляции, leak_score=0.82             |
