# Распределённое оптоволокно — DAS (`fiber`)

**DAS** (Distributed Acoustic Sensing) — это **акустический датчик,
реализованный через оптоволокно**. Тёмные жилы кабеля (unused dark fiber)
работают сенсором на всю длину: лазер посылает импульс, обратно-рассеянное
рэлеевское излучение анализируется на каждом сегменте; вибрации грунта
вокруг кабеля меняют фазу отражённой волны — это и есть акустический сигнал.

**Поверх сырых данных — ML-классификатор слабых сигналов**, который
распознаёт характерные акустические сигнатуры:
- `digging` — экскаваторная копка, ручная копка
- `human_step` — шаги человека
- `noise_event` — катч-олл «нераспознанный громкий шум»
- `vehicle_passing` — проезд тяжёлой техники / автомобиля
- (расширяется обучением модели на новых классах)

UUID источника `3cf6f24b-2dda-4657-92af-43cfb42d3c2e` в СИГМЕ — это и есть
такая DAS-система мониторинга на 10-километровой трассе. На май 2026 в системе
зарегистрировано 503 события этого типа (см. [../../sigma-snapshot-2026-05-17.json](../../sigma-snapshot-2026-05-17.json)).

### Почему отдельный тип от `noise/`

| Свойство          | `noise` (точечный)             | `fiber` (DAS)                          |
|-------------------|--------------------------------|----------------------------------------|
| Физика            | Микрофон / гидрофон в точке     | Кабель = сенсор на всю длину           |
| Координата        | Фиксированная (где висит)       | Динамическая (вдоль кабеля, `x`)        |
| Выход             | Непрерывный (дБ / спектр)       | Триггерный (категория + confidence)     |
| Покрытие          | 1 точка                         | До 50 км одним интеррогатором          |
| Применение        | Поиск утечек в ИТП               | Охрана периметра, защита трасс         |

## Прибор

- Оптоволоконный кабель (одномодовый), уложенный вдоль охраняемой трассы
- DAS-интеррогатор на конце кабеля (Febus, OptaSense, Silixa и др.)
- Edge-сервер с ML-моделью (CNN/трансформер на 30-сек окнах сигнала)
- Опрос непрерывный, события порождаются только при классификации > порога

## Поля payload

| Поле          | Тип    | Описание                                                  |
|---------------|--------|-----------------------------------------------------------|
| `event`       | string | Категория: `digging` / `human_step` / `noise_event` / …    |
| `x`           | number | Координата вдоль волокна (метры от нулевой точки)          |
| `confidence`  | number | Уверенность ML-классификатора (0..1)                       |

Координата только **одна** — `x` (нет `y`). Кабель линейный, событие
локализуется по времени пробега обратно-отражённого импульса.

## Поведение

Триггерный, как и [`video-detector/`](../video-detector/) — событие
рождается только при классификации > порога. Норма ≈ «тишина» (события
не приходят). Это отличает DAS от телеметрических `p`/`t`/`flow`, где
поток данных идёт всегда.
