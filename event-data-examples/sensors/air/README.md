# Качество воздуха (`air`)

Датчики содержания загрязнителей в атмосфере — CO2, угарный газ (CO),
NO2, твёрдые частицы PM2.5 / PM10. Типовой компонент проекта
**ВОЗДУХ-ОМ** (подсистема системы ГОРОД-ОМ-ИИ, см. отчёт ЦИИНГУ-2024
«Шаблон событий», §2 и §6).

Шаблоны и пороги в этой папке взяты из того отчёта дословно — это
формальный контракт обмена между ВОЗДУХ-ОМ и СИГМОЙ.

## Принцип «edge-детектор, не сырая телеметрия»

Датчик CO2 на стороне устройства работает раз в N секунд и шлёт
числовой замер. **В СИГМУ эти замеры по факту не идут** — иначе
центральный брокер событий лёг бы под нагрузкой при тысячах датчиков
по городу (см. отчёт §2 «Рекомендации по масштабируемости» и §3
«Агрегирование данных на местах»).

Вместо этого edge-агрегатор имеет **собственный детектор превышения
норм** и шлёт событие только когда концентрация перешла порог. Это —
**семантический ивент**, а не телеметрия.

```
датчики 1 Гц → edge-агрегатор (district-level) → СИГМА (только при превышении)
   тысячи замеров                                   ~1 ивент в час на узел
```

## Прибор

- NDIR-сенсор CO2 / СО (инфракрасный), точность ±50 ppm
- Лазерный нефелометр для PM2.5 / PM10
- Электрохимический сенсор NO2 / SO2
- Edge-устройство собирает 1-сек замеры, усредняет окном 1–5 мин,
  сравнивает с порогом — отправляет событие только при превышении

## Поля payload

Поля mirror'ят шаблон из отчёта ([§5 п.5](../../README.md) — пример для
загрязнения воздуха):

| Поле           | Тип     | Описание                                                       |
|----------------|---------|---------------------------------------------------------------|
| `event_type`   | string  | Категория события: `air_pollution`, `co2_threshold_exceeded`   |
| `sensor_id`    | string  | ID источника (см. отчёт §2 «Регистрация координат»)            |
| `pollutant`    | string  | `CO2` / `PM2.5` / `PM10` / `NO2` / `CO` / `SO2`                |
| `concentration`| number  | Текущая концентрация                                            |
| `threshold`    | number  | Норма ПДК, с которой сравнили (отчёт §4 п.5)                    |
| `unit`         | string  | `ppm` (CO2/CO/NO2) или `µg/m³` (PM2.5/PM10)                     |
| `ts`           | int64   | Unix-timestamp в миллисекундах (формат из отчёта)               |
| `pos`          | string  | `"<lat>N <lon>E"` (формат из отчёта)                            |

## Типичные пороги ПДК

| Загрязнитель | Норма (рабочая зона / помещение) | Источник нормы            |
|--------------|----------------------------------|---------------------------|
| CO2          | 1000 ppm                          | СанПиН 1.2.3685-21         |
| PM2.5        | 35 µg/m³ (24-часовая средняя)     | US EPA NAAQS              |
| PM10         | 60 µg/m³ (24-часовая средняя)     | СанПиН 1.2.3685-21         |
| NO2          | 0.2 mg/m³ среднесуточная          | ГН 2.1.6.3492-17           |
| CO           | 3 mg/m³ среднесуточная            | ГН 2.1.6.3492-17           |
